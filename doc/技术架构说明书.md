> **版本**: 2.0  
> **最后更新**: 2026-07-12  
> **适用范围**: 全项目（CLI + Web UI + 打包分发）

---

## 1. 系统概述

### 1.1 项目定位

对英语仿读音频进行**语音特征**（OpenSMILE）和**文字准确性**（Whisper + LLM）双重评估，输出多维度评分、结构化差异报告和错题可视化。

### 1.2 核心能力

| 能力 | 技术实现 |
|------|---------|
| 语音声学分析 | OpenSMILE ComParE 2016 LLD，8 维度评分（基频、能量、Jitter、Shimmer、HNR、谱质心、语速、综合分） |
| 语音转文字 | OpenAI Whisper (small.en / medium.en)，支持 CPU/OpenVINO 推理 |
| 文本比对 | DeepSeek API (v4-pro 思考模式)，逐句标注替换/多读/漏读错误 |
| 知识点关联 | 从 `learning_source.csv` 查询错误单词的学习资料链接 |
| 错题可视化 | matplotlib + wordcloud：三分类词云 + 历史进步曲线 |
| Web 管理界面 | Vue 3 SPA + FastAPI 后端，班级/单元管理，实时进度监控 |
| 配置管理 | YAML + .env 双重配置，UI 配置中心可视化编辑 |
| 断点续传 | progress.json 状态机，中断后自动跳过已完成步骤 |

## 3. 系统架构

### 3.1 分层架构

```
┌─────────────────────────────────────────────────┐
│                   ui/ (Vue 3 SPA)                │
│            HTTP REST + SSE EventSource           │
├─────────────────────────────────────────────────┤
│               server/ (FastAPI)                  │
│   REST: /api/v1/{classes,files,tasks,results,…} │
│   SSE:  /events/tasks/{task_id}                 │
├─────────────────────────────────────────────────┤
│             services/ (业务逻辑层)               │
│   class / file / config / pipeline / result /    │
│   log / app_context                              │
├─────────────────────────────────────────────────┤
│                src/ (核心引擎)                    │
│   launcher → voice_compare / text_llm /          │
│   audio_output / filter_name / error_visualizer  │
├─────────────────────────────────────────────────┤
│              resource/ (数据层)                   │
│   classes/  units/  config.yaml  .env  models/   │
└─────────────────────────────────────────────────┘
```

### 3.2 依赖方向（单向，不可逆）

```
ui/ ──── HTTP/SSE ────▶ server/ ────▶ services/ ────▶ src/
                              │              │              │
                              └──────────────┴──────────────┘
                                   均不反向依赖上层
```

**零侵入原则**：`server/`、`services/`、`ui/` 是新增层，通过薄封装调用 `src/` 现有 API，不修改 `src/` 内部逻辑。

### 3.3 数据模型

```
resource/
├── units/{单元名}/                    # 共享教材（所有班级可见）
│   ├── standard_audio/                # 标准音频（1 个 .wav/.mp3）
│   └── standard_text/                 # 标准文本（1 个 .txt）
├── classes/{班级名}/                  # 班级私有数据
│   ├── students.csv                   # 学生名单（name, student_id）
│   └── {单元名}/                      # 本班级在本单元的数据
│       ├── imitation_audio/           # 学生仿读音频
│       └── result/                    # 评估结果
│           ├── progress.json          # 进度状态机
│           ├── summary.csv            # 汇总成绩表
│           ├── {学生名}/              # 单生结果
│           │   ├── {学生名}.md        # LLM 比对报告
│           │   ├── {学生名}_errors.json
│           │   └── voice_comparison_report.png
│           ├── error_analysis/        # 错题可视化输出
│           └── history/               # 历史归档
└── config.yaml                        # 全局配置
```

## 4. 核心模块详解

### 4.1 `src/launcher.py` — 主调度器

```
8 阶段流水线:
  Phase 1: 标准文件发现 + 特征预计算
  Phase 2: 语音分析（OpenSMILE，可关闭）
  Phase 3: Whisper 转写 + LLM 比对（可分别关闭）
  Phase 4: 等待所有任务完成
  Phase 5: 后处理（Excel 汇总）
  Phase 6: 最终汇总 + 耗时统计
  Phase 7: 数据完整性预检查
  Phase 8: 归档 + 错题可视化
```

**并发模型**：三线程池分离调度
- `voice_executor` (1 worker) — C++ 库非线程安全，强制串行
- `whisper_executor` (1 worker) — PyTorch 全局单例，强制串行
- `llm_executor` (400 workers) — 纯网络 I/O，高并发

**断点续传**：`progress.json` 状态机
```
pending → voice:running → voice:done/failed
pending → text:running → text:transcribed → text:done/failed
```

### 4.2 `services/pipeline_service.py` — 任务生命周期管理

- 后台线程调用 `launcher.main()`，通过 `contextlib.redirect_stdout` 捕获日志
- 结构化进度事件（`student_progress`、`stage_progress`、`student_scan`）经 `progress_cb` 回调推送
- 同时解析日志行模式（`[Voice]`、`[LLM]`、`[Phase X]`）作为补充进度来源
- SSE 推送通过 `asyncio.Queue` 实现跨线程安全的事件驱动
- 强制终止：`cancel_event.set()` + `ThreadPoolExecutor.shutdown(wait=False, cancel_futures=True)` + 幂等终态标记

### 4.3 `server/events/sse.py` — 实时事件流

```
Client ──GET /events/tasks/{id}──▶ SSE Stream
                                      │
                          ┌───────────┴───────────┐
                          │  asyncio.Queue (线程安全) │
                          └───────────┬───────────┘
                                      │
                    PipelineService._push_event()
                    (loop.call_soon_threadsafe)
```

- 首次连接发送完整快照，后续增量推送
- 15s 心跳保持连接
- 终态（completed/failed/cancelled/not_found）自动关闭流

### 4.4 `src/text_llm.py` — LLM 调用层

- `call_llm()`: 支持系统提示词 + 用户提示词，可选 JSON 模式
- 最多 3 次重试 + 指数退避（1.5s → 3s），捕获 `IncompleteRead`、`RemoteDisconnected` 等网络异常
- DeepSeek 思考模式：token 耗尽时自动回退到非思考模式
- `llm_compare_texts()`: 逐句比对，输出结构化错误数据（单词级）+ 知识点链接

### 4.5 `ui/src/App.vue` — 前端 SPA

**8 个页面视图**：
- 仪表盘：系统状态卡 + 成绩分布图 + 任务控制
- 教材管理：单元 CRUD + 标准音频/文本上传
- 班级管理：班级 CRUD + 学生 CSV 导入/手动添加
- 音频管理：单/批量上传，提交状态 + 评分状态一览
- 结果总览：成绩表排序 + 学生详情 Modal（Markdown 渲染）
- 错误分析：四分类词云图 + 错误词频表
- 数据导出：按班级/单元文件下载
- 配置中心：LLM/Whisper/模块开关可视化编辑

## 7. 打包与分发

### 7.1 打包方式

使用 `build_exe.py` 脚本，底层调用 PyInstaller（onedir 模式）：

```bash
python build_exe.py          # 生产模式（含模型文件）
python build_exe.py --dev    # 开发模式（不含模型，约 200MB）
```

### 7.2 分发结构

```
voice_analyse/
├── voice_analyse.exe        # 入口
├── _internal/               # PyInstaller 依赖
├── models/                  # Whisper 模型（生产模式）
├── resource/                # 配置 + 数据
├── ui/dist/                 # 前端
├── doc/                     # 文档
└── 使用说明.txt
```

### 7.3 目标环境

- Windows 10/11 64-bit
- Python 3.10+ 预装（含 PyTorch、Whisper、OpenSMILE）
- FFmpeg 预装

## 8. 开发规范

### 8.1 Python

- 所有模块顶部中文 docstring（功能、依赖）
- 私有函数 `_` 前缀；线程锁 `_xxx_lock` 后缀
- `matplotlib.use("Agg")` 必须在任何 matplotlib 导入前设置
- 单学生失败标记 `failed`，不阻塞其他学生

### 8.2 Vue/TypeScript

- Composition API + `<script setup lang="ts">`
- 所有 API 类型定义在 `ui/src/types/index.ts`，禁止 `any`
- 每个数据获取有 loading/error/empty 三态处理

### 8.3 Git

- `main` — 稳定发布分支
- `dev` — 日常开发分支
- 提交信息格式：`feat: / fix: / docs: / refactor:`

## 9. 已知限制

| 限制 | 说明 |
|------|------|
| 单机运行 | 不支持分布式，多用户需各自部署 |
| Windows 优先 | 路径分隔符、字体等以 Windows 为基准 |
| 内存敏感 | Whisper + PyTorch 需 ≥4GB RAM |
| LLM 速率限制 | 并发过高可能触发 API 限流 |
| 无用户认证 | 本地单用户模式，无登录机制 |
| 大文件耗时 | 长音频的 Whisper 转写可能 >30s/条 |
