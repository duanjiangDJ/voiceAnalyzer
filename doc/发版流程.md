# 发版流程

## 前置检查

- [ ] 所有代码更改已合并到 `dev` 分支
- [ ] Vue 前端已构建 (`cd ui && npm run build`)
- [ ] `python run.py` 命令行模式验证通过
- [ ] `python run_ui.py` Web UI 模式验证通过
- [ ] 单元测试通过（如有）
- [ ] `resource/config.yaml` 为默认配置（不含真实密钥）

## 发版步骤

### 1. 版本号更新

在以下文件中更新版本号：
- `resource/config.yaml` → `app.version`
- `ui/package.json` → `version`
- `readme.md` → 标题或徽章

### 2. 开发分支合并

```bash
git checkout dev
git pull origin dev
git checkout main
git merge dev
git tag vX.Y.Z
git push origin main --tags
```

### 3. 打包（Windows）

```bash
# 安装 PyInstaller
pip install pyinstaller

# 生产模式打包（包含模型文件，体积大）
python build_exe.py

# 开发模式打包（不含模型，体积小）
python build_exe.py --dev
```

输出目录：`dist/voice_analyse/`

### 4. 分发前验证

- [ ] 解压/复制 `dist/voice_analyse/` 到另一台干净的 Windows 机器
- [ ] 确保目标机器已安装 Python 3.10+ 及依赖（见下方）
- [ ] 双击 `voice_analyse.exe` 确认服务启动
- [ ] 浏览器访问 `http://127.0.0.1:8000` 确认页面正常加载
- [ ] 创建一个测试班级/单元，上传测试音频
- [ ] 启动评估任务，确认正常运行
- [ ] 检查日志、结果、导出功能

### 5. 发布

- [ ] 将 `dist/voice_analyse/` 文件夹打包为 `.zip`
- [ ] 命名格式：`voice_analyse_vX.Y.Z_windows.zip`
- [ ] 上传到 GitHub Releases
- [ ] 编写 Release Notes（参考下方模板）

## 目标环境依赖

分发的 exe 依赖用户系统已安装以下组件：

| 组件 | 安装方式 | 说明 |
|------|---------|------|
| Python 3.10+ | 官网/conda | exe 实际调用 Python 解释器 |
| PyTorch | `pip install torch` | Whisper 模型推理 |
| OpenAI Whisper | `pip install openai-whisper` | 语音转写 |
| OpenSMILE | `pip install opensmile` | 声学特征提取 |
| FFmpeg | `choco install ffmpeg` / 官网 | 音频解码 |

## 回滚指南

若发版后发现严重问题：

```bash
git checkout main
git revert <bad-commit-hash>
git push origin main
```

或直接切回上一个稳定 tag：

```bash
git checkout vX.Y.Z-1
```
